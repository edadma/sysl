## Primary source locations

- `trisc/`: language reference materials and supporting project docs
- `trisc/sysl/`: SysL compiler/interpreter code

## Documentation conventions

- Keep normative language behavior documented under `/reference`.
- Keep implementation behavior documented under `/implementation`.
- When behavior changes in code, update both reference and implementation docs as needed.
