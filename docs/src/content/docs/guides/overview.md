SysL documentation is organized around two repositories paths:

- `trisc/` for language-level specification and references
- `trisc/sysl/` for compiler/interpreter implementation details

This docs site should mirror that split:

- Put syntax, grammar, and semantic behavior in **Language Reference**
- Put parser/compiler/runtime internals in **Implementation**
